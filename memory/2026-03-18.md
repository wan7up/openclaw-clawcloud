- QMD / memory_search 关键结论：这次问题不是单一配置错，而是分阶段暴露：先是 embeddings 错走 OpenAI provider（用中转 API key 打官方 embeddings 导致 401），再是 collection 命名不一致（`memory-root-main` vs `memory-root`），再是本地依赖缺失（`node-llama-cpp` 为 optional peer，需显式装入 OpenClaw 运行目录），最后在 2G 机器上卡在本地 query 性能瓶颈；升到 8G 后，`memory_search` 已恢复可用，说明软件链路已基本打通，剩余主要是 QMD 原生 query 路径偏重的性能问题。
- 经验：修 OpenClaw 本地记忆检索时，区分三层问题很重要——(1) provider / key / baseUrl 是否走对；(2) collection 命名与索引配置是否一致；(3) 本地 embeddings 依赖和机器资源是否足够。不要一开始就把所有慢都归咎于机器性能。
