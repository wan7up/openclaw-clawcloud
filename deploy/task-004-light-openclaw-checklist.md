# 任务 004：轻量 OpenClaw 部署说明（1GB 内存设备）

## 目标形态

适用于：
- 4 核 ARM Cortex-A53
- 1GB RAM（可开少量 swap）
- 主渠道：飞书
- 模型：API 调用
- 本地能力：局域网打印、局域网存储

核心目标：
- 能稳定收发消息
- 能调用模型回复
- 能访问打印机与局域网存储
- 尽量降低内存占用与后台复杂度

---

## 保留的功能

### 必留
- Feishu 主渠道
- API 模型调用
- 基础聊天能力
- 必要的文件处理能力
- 打印相关系统能力（如 CUPS / lp）
- 局域网存储访问能力（SMB / NFS / 挂载目录）
- 基本记忆与任务文件（workspace / memory / TASKS.md）

### 可留但要克制
- 少量定时任务
- 少量文档处理
- 基础文件回传

---

## 建议禁用 / 不启用的功能

### 建议禁用
- 多渠道同时接入（先只保留飞书）
- 浏览器自动化
- 大量子任务 / 并发任务
- 重度文件批处理
- 不必要的实时流式特效
- 群聊大范围开放接入
- 重型插件 / 不会用到的渠道插件配置

### 不适合这台机器的场景
- 高频并发聊天
- 重度网页操作
- 本地大模型推理
- 大量长上下文复杂任务
- 多人同时高频使用

---

## 配置原则

### 渠道
- 只开 Feishu
- 其他渠道先不配置或禁用

### 模型
- 只走远端 API
- 不做本地模型

### 接入策略
- 优先白名单/有限开放
- 不要一开始就全员高并发开放

### 工具
- 只保留你会用到的工具链
- 打印走系统命令
- 存储走挂载目录/局域网共享

### 系统层
- 可配置少量 swap，但不要指望 swap 代替真内存
- 尽量减少其他常驻进程
- 日志不要无限膨胀

---

## 部署后自检清单

### 一、基础状态
- [ ] OpenClaw 能正常启动
- [ ] Feishu 能正常收消息
- [ ] Feishu 能正常回复消息
- [ ] 模型调用正常

### 二、内存与稳定性
- [ ] 空闲时内存占用可接受
- [ ] 连续聊天后不会频繁 OOM
- [ ] 开启 swap 后系统没有明显卡死

### 三、局域网能力
- [ ] 能访问局域网打印机
- [ ] 能执行打印命令
- [ ] 能访问局域网存储目录
- [ ] 读写目标目录正常

### 四、功能边界
- [ ] 未启用不必要渠道
- [ ] 未启用浏览器自动化
- [ ] 未启用高并发/重后台任务
- [ ] 群聊或多用户访问范围受控

### 五、恢复与维护
- [ ] workspace 已纳入备份
- [ ] ~/.openclaw 已纳入备份
- [ ] 备份方案可用

---

## 后续优化顺序

如果后面发现资源紧张，按这个顺序收缩：
1. 先减少渠道和并发
2. 再减少重工具/重任务
3. 再限制大文件处理
4. 最后才考虑更激进的定制瘦身

---

## 结论

这台设备适合跑一个“飞书主渠道 + API 模型 + 局域网打印/存储”的轻量 OpenClaw。
重点不是做新包，而是做一套克制的配置，并在部署后按本清单逐项检查。
