# 任务 008：OpenClaw 跨服务器备份与恢复实施方案

## 目标

在服务器不稳定、需要随时迁移到另一台服务器时，尽可能完整地恢复 OpenClaw：

- 恢复人格/记忆/任务/技能
- 恢复配置
- 尽量恢复 OAuth / 渠道 / 会话 / 定时任务等状态
- 异地备份与日常文件隔离，避免误删主盘文件

## 备份分层

### A. 核心层：workspace
通常路径：
- `~/.openclaw/workspace/`

价值：
- AGENTS / SOUL / USER / IDENTITY / TASKS
- memory/
- 自定义技能、文档、工作文件

### B. 状态层：整份 state dir
通常路径：
- `~/.openclaw/`

价值：
- `openclaw.json`
- `agents/<agent>/agent/auth-profiles.json`
- `agents/<agent>/sessions/`
- `credentials/`
- `cron/`
- `devices/`
- `telegram/`
- 其他运行状态

## 推荐异地备份目标

### 最推荐
- 专用备份账号 / 专用 bucket / 专用目录
- 只用于 OpenClaw 备份
- 不混放日常文件

### 权限建议
- 尽量限制在专用路径内
- 优先启用版本保留 / 生命周期
- 如后端支持，开启对象版本化或回收站

## 实施顺序

### 第一步：确定远端备份目标（你来）
你需要准备下面三种之一：

1. **专用对象存储 / bucket**（最推荐）
2. **专用网盘目录/账号，通过 rclone 接入**
3. **另一台服务器/NAS 的专用备份目录**

### 第二步：配置 rclone（你来）
在服务器上完成：
- `rclone config`
- 建立一个 remote，例如 `ocbackup:`
- 确保 remote 指向专用备份空间

### 第三步：我来落备份脚本
脚本职责：
- 停 gateway（可选，做一致性快照时使用）
- 打包 `~/.openclaw`
- 生成校验信息
- 上传到远端备份目标
- 保留本地最近若干份

### 第四步：你来执行首次人工备份
确认：
- 上传成功
- 远端能看到文件
- 文件大小正常

### 第五步：我来补自动化
可选：
- cron 定时
- 每日 / 每周策略
- 保留规则

### 第六步：恢复演练（你有空时做一次）
在新服务器上：
1. 安装 OpenClaw
2. 恢复备份
3. 运行 `openclaw doctor`
4. 启动 gateway
5. 检查模型、配置、任务、记忆是否恢复

## 需要你配合的关键步骤

### 你必须提供/完成
1. 备份后端选型
2. rclone remote 配置
3. 备份目标凭证
4. 首次备份后的结果确认
5. 恢复演练时的新服务器环境

### 我可以先做
1. 备份脚本
2. 恢复说明
3. 目录结构约定
4. 保留策略建议

## 推荐默认方案

如果你现在想先走最稳妥路线：

- 用 **rclone + 专用备份目标**
- 备份整份 `~/.openclaw`
- 不碰你的主网盘其他目录
- 后续再加 cron
